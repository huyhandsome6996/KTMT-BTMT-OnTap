Bài 8: CRC (Cyclic Redundancy Check) — Phát hiện lỗi bằng đa thức

CRC là "vệ sĩ" của dữ liệu: khi bạn ghi file vào USB, tải file ZIP, hay gửi gói tin mạng — CRC âm thầm kiểm tra xem dữ liệu có bị hỏng không. Học kỹ phần này, đề thi chắc chắn có!

📖 Lý thuyết

🔑 Kiến thức cốt lõi

CRC (Cyclic Redundancy Check) là phương pháp phát hiện lỗi dựa trên phép chia đa thức trong trường GF(2). Ý tưởng: gửi kèm dữ liệu một đoạn "mã dư" (CRC) được tính từ dữ liệu. Khi nhận, chia lại — nếu dư = 0 thì dữ liệu nguyên vẹn, dư ≠ 0 là bị lỗi.

Ứng dụng: kiểm tra khi đọc/ghi đĩa (HDD, SSD, USB), nén file (ZIP, RAR), giao thức mạng (Ethernet, Wi-Fi, USB, Bluetooth).

1. Phép XOR — "Cộng trừ" trong thế giới bit

Trong trường GF(2) (chỉ có 2 giá trị 0 và 1), cộng = trừ = XOR. Không có carry, không có borrow. Đơn giản là "cùng nhau = 0, khác nhau = 1".

A B A ⊕ B (XOR) Giải thích
000cùng nhau (đều 0) → 0
011khác nhau → 1
101khác nhau → 1
110cùng nhau (đều 1) → 0
💡 Mẹo nhớ XOR

Nghĩ XOR là câu hỏi: "Hai bit có giống nhau không?" — Giống → 0, Khác → 1. Hoặc: XOR = "cộng bỏ carry" (1+1 = 10, bỏ 1 carry chỉ còn 0).

2. Đa thức sinh G(x) — "Thước đo" để chia

📐 Đa thức sinh G(x)

G(x) là đa thức bậc n, dùng làm "số chia" trong phép chia CRC. Quy ước:

  • Mỗi số hạng $x^k$ trong đa thức tương ứng 1 bit 1 ở vị trí $k$.
  • Dạng nhị phân của G(x) có (n+1) bit (bậc n + bit 0).
  • Bit cao nhất (MSB) luôn = 1 (vì bậc cao nhất luôn có mặt).

Ví dụ: $G(x) = x^4 + x + 1$ → bit ở vị trí 4, 1, 0 là 1; còn lại 0 → 10011 (5 bit, bậc 4).

$$G(x) = x^4 + x + 1 \;\Rightarrow\; \text{binary} = 1\cdot x^4 + 0\cdot x^3 + 0\cdot x^2 + 1\cdot x^1 + 1\cdot x^0 = 10011$$

3. Quy trình 6 bước tính CRC

Bước 1: Cho thông điệp M(x) (dạng nhị phân, k bit). Bước 2: Cho đa thức sinh G(x) bậc n (dạng nhị phân, n+1 bit, MSB = 1). Bước 3: Tạo N(x) = M(x) · x^n ⇔ N = M + (n số 0 phía sau). (Thêm n bit 0 vào cuối M để "chừa chỗ" cho CRC). Bước 4: Chia N(x) cho G(x) bằng XOR (chia mod-2): - Lấy (n+1) bit đầu của N làm "thanh ghi". - Nếu MSB = 1: XOR thanh ghi với G. - Nếu MSB = 0: KHÔNG XOR (giữ nguyên). - Bỏ bit MSB (= 0 sau XOR), hạ 1 bit tiếp theo từ N. - Lặp lại đến khi hết bit của N. Bước 5: Phần dư sau chia = CRC (đúng n bit). Bước 6: Dữ liệu truyền đi = M(x) + CRC (ghép CRC vào sau M).

4. Các đa thức sinh chuẩn (thường dùng)

Tên Đa thức G(x) Dạng nhị phân Bậc Ứng dụng
CRC-12 $x^{12} + x^{11} + x^3 + x^2 + x + 1$ 1100000001111 12 Telecom (ký tự 6 bit)
CRC-16 (ANSI) $x^{16} + x^{15} + x^2 + 1$ 11000000000000101 16 IBM Bisync, Modbus
CRC-CCITT $x^{16} + x^{12} + x^5 + 1$ 10001000000100001 16 X.25, HDLC, Bluetooth
CRC-32 $x^{32} + x^{26} + x^{23} + \ldots + 1$ 100000100110000010001110110110111 32 Ethernet, ZIP, RAR, PNG
⚠️ Điểm dễ sai
  • Quên thêm n bit 0 vào sau M trước khi chia. Nếu quên, kết quả CRC sẽ sai hoàn toàn.
  • Nhầm "bậc n" với "số bit n+1". G(x) bậc 4 → có 5 bit, CRC có 4 bit, thêm 4 số 0.
  • XOR sai vì dùng phép cộng thường (1+1=2 thay vì 1⊕1=0). Nhớ: trong CRC, KHÔNG có carry!
  • Quên bỏ bit MSB = 0 sau mỗi lần XOR. Phải bỏ rồi mới hạ bit tiếp theo.
  • Dừng sai: phải chia đến khi nào hết bit của N mới dừng, không phải khi thanh ghi = 0.

📝 Bài 1

Tính CRC cho M = 1100 (4 bit) với G(x) = x³ + x + 1 = 1011. Tìm dữ liệu truyền đi.

📌 Công thức áp dụng: Bậc G = n = 3 → N = M + 3 bit 0. Chia N cho G bằng XOR (mod-2). CRC = phần dư (n bit). Dữ liệu truyền = M + CRC. XOR: 1⊕1=0, 1⊕0=1, 0⊕0=0, 0⊕1=1.

💡 Cách áp dụng: Khởi tạo thanh ghi = 4 bit đầu của N. Mỗi bước: xem MSB — nếu =1 thì XOR với G, nếu =0 thì bỏ qua. Bỏ bit đầu, hạ 1 bit từ N xuống. Lặp đến khi hết N. Dư cuối = CRC.

🧮 Tính toán từng bước:

Bước 0 — Chuẩn bị: G(x) = 1011 (4 bit, bậc n = 3) N = M + 3 bit 0 = 1100 + 000 = 1100000 (7 bit) → Sẽ có 7 − 4 + 1 = 4 bước XOR Bước 1: working = 1100 (4 bit đầu của N) MSB = 1 → XOR với G = 1011: 1100 ⊕ 1011 ───── 0111 Bỏ bit đầu (=0) → dư tạm = 111 Bit thương Q = 1 Bước 2: hạ N[4]=0 → working = 1110 MSB = 1 → XOR với 1011: 1110 ⊕ 1011 ───── 0101 Dư tạm = 101 Q = 1 Bước 3: hạ N[5]=0 → working = 1010 MSB = 1 → XOR với 1011: 1010 ⊕ 1011 ───── 0001 Dư tạm = 001 Q = 1 Bước 4: hạ N[6]=0 → working = 0010 MSB = 0 → KHÔNG XOR (giữ nguyên) Dư tạm = 010 Q = 0 → Dư cuối = CRC = 010 (3 bit, đúng bằng bậc của G) → Thương Q = 1110 (không cần quan tâm)

✓ Kết quả: CRC = 010, dữ liệu truyền = M + CRC = 1100 + 010 = 1100010

🔍 Kiểm tra (chia dữ liệu truyền cho G, dư phải = 0):

Chia 1100010 cho 1011: B1: 1100 ⊕ 1011 = 0111 → dư 111 B2: hạ 0 → 1110 ⊕ 1011 = 0101 → dư 101 B3: hạ 1 → 1011 ⊕ 1011 = 0000 → dư 000 B4: hạ 0 → 0000 (MSB=0, không XOR) → dư 000 → Dư = 000 ✓ (chia hết, dữ liệu nguyên vẹn)
📝 Bài 2

Tính CRC cho M = 101010 (6 bit) với G(x) = x³ + x² + 1 = 1101. Tìm dữ liệu truyền đi.

📌 Công thức áp dụng: N = M + n bit 0 (n = bậc G). Chia N cho G bằng XOR. CRC = phần dư. Dữ liệu truyền = M || CRC.

💡 Cách áp dụng: G = 1101 (bậc 3) → thêm 3 bit 0. N có 9 bit → 9−4+1 = 6 bước XOR. Mỗi bước: MSB=1 XOR, MSB=0 bỏ qua.

🧮 Tính toán từng bước:

Bước 0 — Chuẩn bị: G = 1101 (4 bit, bậc n = 3) N = 101010 + 000 = 101010000 (9 bit) → 9 − 4 + 1 = 6 bước XOR Bước 1: working = 1010 MSB=1 → XOR 1101: 1010 ⊕ 1101 = 0111 → dư = 111, Q=1 Bước 2: hạ N[4]=1 → working = 1111 MSB=1 → XOR 1101: 1111 ⊕ 1101 = 0010 → dư = 010, Q=1 Bước 3: hạ N[5]=0 → working = 0100 MSB=0 → KHÔNG XOR → dư = 100, Q=0 Bước 4: hạ N[6]=0 → working = 1000 MSB=1 → XOR 1101: 1000 ⊕ 1101 = 0101 → dư = 101, Q=1 Bước 5: hạ N[7]=0 → working = 1010 MSB=1 → XOR 1101: 1010 ⊕ 1101 = 0111 → dư = 111, Q=1 Bước 6: hạ N[8]=0 → working = 1110 MSB=1 → XOR 1101: 1110 ⊕ 1101 = 0011 → dư = 011, Q=1 → Dư cuối = CRC = 011 (3 bit) → Thương Q = 110111

✓ Kết quả: CRC = 011, dữ liệu truyền = M + CRC = 101010 + 011 = 101010011

🔍 Kiểm tra (chia dữ liệu truyền cho G):

Chia 101010011 cho 1101: B1: 1010 ⊕ 1101 = 0111 → dư 111 B2: hạ 1 → 1111 ⊕ 1101 = 0010 → dư 010 B3: hạ 0 → 0100 (MSB=0, skip) → dư 100 B4: hạ 0 → 1000 ⊕ 1101 = 0101 → dư 101 B5: hạ 1 → 1011 ⊕ 1101 = 0110 → dư 110 B6: hạ 1 → 1101 ⊕ 1101 = 0000 → dư 000 → Dư = 000 ✓ (chia hết, dữ liệu nguyên vẹn)
📝 Bài 3

Bên nhận nhận được 101011010110, đa thức sinh G = 1101. Dữ liệu có bị lỗi không?

📌 Công thức áp dụng: Tại đầu nhận, chia dữ liệu nhận cho G(x). Dư = 0 → không lỗi. Dư ≠ 0 → có lỗi. (Không cần biết M gốc, chỉ cần G.)

💡 Cách áp dụng: Dữ liệu 12 bit, G 4 bit → 12 − 4 + 1 = 9 bước XOR. Tính phần dư cuối. Nếu dư = 000 → dữ liệu nguyên vẹn, ngược lại → bị lỗi.

🧮 Tính toán từng bước:

Bước 0 — Dữ liệu nhận: 101011010110 (12 bit), G = 1101 → 12 − 4 + 1 = 9 bước XOR Bước 1: working = 1010, MSB=1 1010 ⊕ 1101 = 0111 → dư = 111, Q=1 Bước 2: hạ 1 → 1111, MSB=1 1111 ⊕ 1101 = 0010 → dư = 010, Q=1 Bước 3: hạ 1 → 0101, MSB=0 → skip → dư = 101, Q=0 Bước 4: hạ 0 → 1010, MSB=1 1010 ⊕ 1101 = 0111 → dư = 111, Q=1 Bước 5: hạ 1 → 1111, MSB=1 1111 ⊕ 1101 = 0010 → dư = 010, Q=1 Bước 6: hạ 0 → 0100, MSB=0 → skip → dư = 100, Q=0 Bước 7: hạ 1 → 1001, MSB=1 1001 ⊕ 1101 = 0100 → dư = 100, Q=1 Bước 8: hạ 1 → 1001, MSB=1 1001 ⊕ 1101 = 0100 → dư = 100, Q=1 Bước 9: hạ 0 → 1000, MSB=1 1000 ⊕ 1101 = 0101 → dư = 101, Q=1 → Dư cuối = 101 ≠ 000

✓ Kết quả: Phần dư = 101 ≠ 000 → CÓ LỖI! Dữ liệu bị hỏng trong quá trình truyền.

🔍 Kiểm tra (giải thích vì sao CRC phát hiện được):

CRC có tính chất: dữ liệu truyền (M + CRC) luôn chia hết cho G(x) → dư = 0. Nếu 1+ bit bị đảo trong truyền → chuỗi nhận khác chuỗi gửi → không còn chia hết → dư ≠ 0. Lưu ý: - CRC chỉ PHÁT HIỆN lỗi, không SỬA được. - Để sửa lỗi, dùng mã Hamming (xem Bài 9-10). - Trong thực tế, bên nhận yêu cầu bên gửi truyền lại (ARQ/retransmission). Một số pattern lỗi CRC vẫn không thấy (rất hiếm): - Lỗi mà pattern bit trùng với G(x) hoặc bội của G(x). - Nhưng xác suất rất thấp với G(x) chuẩn (vd CRC-32 → tỉ lệ hụt ~ 2^-32).
📝 Bài 4

Tính CRC cho M = 1111 với G(x) = x⁴ + x + 1 = 10011. Tìm dữ liệu truyền đi.

📌 Công thức áp dụng: N = M + n bit 0 (n = bậc G = 4). Chia N cho G bằng XOR. CRC = dư (4 bit). Dữ liệu truyền = M || CRC.

💡 Cách áp dụng: G = 10011 (5 bit, bậc 4) → thêm 4 bit 0. N có 8 bit → 8 − 5 + 1 = 4 bước XOR. Mỗi bước xem MSB để quyết định XOR hay skip.

🧮 Tính toán từng bước:

Bước 0 — Chuẩn bị: G(x) = x^4 + x + 1 = 10011 (5 bit, bậc n = 4) N = 1111 + 0000 = 11110000 (8 bit) → 8 − 5 + 1 = 4 bước XOR Bước 1: working = 11110 MSB=1 → XOR 10011: 11110 ⊕ 10011 ───── 01101 Dư tạm = 1101, Q=1 Bước 2: hạ N[5]=0 → working = 11010 MSB=1 → XOR 10011: 11010 ⊕ 10011 ───── 01001 Dư tạm = 1001, Q=1 Bước 3: hạ N[6]=0 → working = 10010 MSB=1 → XOR 10011: 10010 ⊕ 10011 ───── 00001 Dư tạm = 0001, Q=1 Bước 4: hạ N[7]=0 → working = 00010 MSB=0 → KHÔNG XOR (giữ nguyên) Dư tạm = 0010, Q=0 → Dư cuối = CRC = 0010 (4 bit, đúng bằng bậc của G) → Thương Q = 1110

✓ Kết quả: CRC = 0010, dữ liệu truyền = M + CRC = 1111 + 0010 = 11110010

🔍 Kiểm tra (chia dữ liệu truyền cho G):

Chia 11110010 cho 10011: B1: 11110 ⊕ 10011 = 01101 → dư 1101 B2: hạ 0 → 11010 ⊕ 10011 = 01001 → dư 1001 B3: hạ 1 → 10011 ⊕ 10011 = 00000 → dư 0000 B4: hạ 0 → 00000 (MSB=0, skip) → dư 0000 → Dư = 0000 ✓ (chia hết, dữ liệu nguyên vẹn) Trick nhớ: G(x) = x^4 + x + 1 = 10011 là CRC-4 ITU chuẩn, hay gặp trong đề thi.

💡 5 bí kíp để được điểm tối đa
  1. Luôn đổi G(x) từ đa thức sang binary trước, ghi rõ "bậc n = ... bit". Vd: $x^4 + x + 1 = 10011$, bậc 4, 5 bit. Nhiều bạn quên bước này → chấm lỗi.
  2. Viết N(x) = M + n số 0 ra rõ ràng trước khi chia. Vd: N = 10010110110111 + 0000 = 100101101101110000. Bit 0 thêm vào tô màu hoặc gạch chân.
  3. Mỗi bước XOR ghi rõ: working ⊕ G = result → dư tạm. Nếu MSB = 0, ghi "bỏ qua" thay vì bỏ trống — giám khảo thấy bạn hiểu quy tắc.
  4. Đánh số bước 1, 2, 3, ... từ trên xuống dưới. Cuối cùng ghi "DƯ CUỐI = ..." in đậm để giám khảo dễ thấy đáp án.
  5. Luôn có dòng "Kiểm tra" ở cuối: chia dữ liệu truyền (M+CRC) cho G, ghi "dư = 0 ✓". Đây là điểm cộng lớn!
⚠️ Những lỗi mất điểm thường gặp
  • ❌ Quên thêm n bit 0 vào sau M trước khi chia → CRC sai hoàn toàn, mất 50% điểm.
  • ❌ Nhầm "bậc n" với "số bit n+1". G bậc 4 → 5 bit, CRC 4 bit, thêm 4 số 0 (KHÔNG phải 5).
  • ❌ Dùng phép cộng thường thay vì XOR: 1+1=2 hoặc viết 1+1=1 (nhầm với OR). Đúng: 1⊕1=0.
  • ❌ Không bỏ bit MSB = 0 sau XOR mà cứ để nguyên rồi hạ bit → sai dài ra.
  • ❌ Dừng quá sớm: phải chia đến khi hết bit của N, không phải khi dư = 0.
  • ❌ Đảo ngược thứ tự bit: CRC luôn ghép vào sau M, không phải trước.
  • ❌ Không ghi "Dữ liệu truyền" cuối bài — chỉ ghi CRC thì mất điểm kết luận.

📌 Tóm tắt cuối bài

✅ Điểm cần nhớ
  1. CRC = phương pháp phát hiện lỗi dựa trên chia đa thức mod-2 (XOR).
  2. XOR: cùng nhau = 0, khác nhau = 1. (1⊕1=0, 1⊕0=1, 0⊕0=0, 0⊕1=1)
  3. G(x) bậc n → có (n+1) bit nhị phân, MSB luôn = 1.
  4. Quy trình 6 bước: M → G → N=M+n số 0 → chia XOR → CRC = dư → truyền M+CRC.
  5. Số bit CRC = bậc n của G (vd G bậc 4 → CRC 4 bit).
  6. Kiểm tra lỗi: chia dữ liệu nhận cho G. Dư = 0 → không lỗi. Dư ≠ 0 → có lỗi.
  7. CRC chỉ phát hiện, không sửa lỗi. Để sửa cần Hamming (Bài 9-10).
  8. Các chuẩn: CRC-12, CRC-16, CRC-CCITT (bậc 16), CRC-32 (bậc 32).
🎯 Mẹo thi nhanh

Khi gặp đề CRC, làm theo trình tự:

1. Đổi G(x) đa thức → binary. Đếm bậc n = (số bit G) - 1. 2. Thêm n số 0 vào sau M → N. 3. Lấy (n+1) bit đầu của N làm thanh ghi. 4. Lặp: MSB=1 → XOR với G; MSB=0 → bỏ qua. Bỏ MSB, hạ 1 bit. 5. Đến khi hết bit N → dư cuối = CRC. 6. Truyền = M || CRC. Kết luận bằng dòng kiểm tra.

Nếu đề cho dữ liệu nhận và hỏi "có lỗi không": chỉ cần chia dữ liệu nhận cho G, xem dư = 0 hay ≠ 0.
